Grid ≠ Tables
PS. Tables aren't bad, they're misunderstood

- No
- CSS grid is specifically designed for creating layouts
- It's deisgned for this use, rather than tables which we used as a hack
- And back to the separation of concerns principle, CSS is where we should be defining how the app/page looks

Grid helps us
write semantic HTML

- CSS Grid also helps us get back to writing logical and semantic HTML
- We're no longer required to write our HTML based on how the page design looks
- We can now write our HTML based on the logical flow of the document and let CSS rearrange it visually
- When done right, this means that our content can now be more accessible to anyone using reader mode, saving and article for later or using a screen reader or assistive technology
- Thanks to CSS Grid, mocking up or building the layout of an app not only takes less time, it takes less code
- And to prove it, today we're going to build a note taking app using CSS Grid
