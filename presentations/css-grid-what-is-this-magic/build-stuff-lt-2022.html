Grid ≠ Tables
PS. Tables aren't bad, they're misunderstood

- No - CSS grid is specifically designed for creating layouts - It's deisgned for this use, rather than tables which we used as a hack - And back to the separation of concerns principle, CSS is where we should be defining how the app/page looks

Grid helps us
write semantic HTML

- CSS Grid also helps us get back to writing logical and semantic HTML - We're no longer required to write our HTML based on how the page design looks - We can now write our HTML based on the logical flow of the document and let CSS rearrange it visually - When done right, this means that our content can now be more accessible to anyone using reader mode, saving and article for later or using a screen reader or assistive technology
- Thanks to CSS Grid, mocking up or building the layout of an app not only takes less time, it takes less code - And to prove it, today we're going to build a note taking app using CSS Grid

- Define that the header should take full width of page - Several ways to do this - One is to define which grid lines it starts and finishes on
- The lines of the grid are defined, and can be used to set where an element sits - If we want the header to start at the beginning, we want it to start at grid line 1 

						header {
							grid-column-start: 1;

- We want it to go all the way to the end - So could set to end at grid line 6 - But if the size of our grid changes, we then need to change that - Grid lines aren't just numbered like this
- They're also numbered backwards - So instead we can define that the header should go to grid line -1 - This means that if the grid changes size, it will always go to the very end - Whether our grid is 1 column wide or 100, the header will continue to take up the full width at the top
